Contactulator 2000

Customer Contact and Order Tracking Database on a Static Webpage

Features Include:

  • Optional Firebase Backend
  • Submit Updates
  • View/Filter Customer Information
  • Query Recent Orders
  • Sort/Filter Order Information
  • Does not require SQL database
  • Allows users to store master database offline

The Contactulator 2000 is a Jekyll driven "database" with an optional Firebase backend. Thanks to Jeklly it functions on a static webpage and does not require a traditional database to operate instead it utilizes relies solely on the Jekyll templating engine and JSON dictionaries. User can query customer contacts or employees, view orders and account information, all on a static webpage. Updating the system is as easy as saving a JSON formatted file in the data folder. This design allows users to keep thier valuable customer and order information offline away from online threats and hackers. To speed up the updating process I created a webform and configured a Firebase database to accept updates from users which can be downloaded later and used to update the system. The Contactulator 2000 is a barebones proof of concept project that can be expanded upon and customized to meet database needs of any small business.

Re-Action To-Do List

Website Displaying Useful & Not So Useful APIs

Features Include:

  • React Frontend
  • Create and Delete Notes
  • FireBase Backend
  • Changes Persist Through Page Reloads

Re-Action To-Do List Generator is a fully functional list creator. This application utilizes React and Firebase and is a great example of how JavaScript can be persistent through page reloads when combined with a FireBase database. Yes you read that right, changes to the DOM are remain even after the page reloads. This feat is accomplished by exporting the user's input to FireBase and then importing the data everytime the page reloads. If you are looking for a great example of how to create and delete items in a FireBase database using React check out Re-Action today.
